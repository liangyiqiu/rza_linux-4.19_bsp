Device IDs

Hardware components

The lowlevel hardware control is mainly based on direct hardware register access.  Each component has its own register field.  Registers are not accessed directly by an address, but rather by a device ID and an index (see d1_setregister or d1_getregister).

The driver will map the abstract register index to a platform dependant physical register address.  Not every platform exposes the same DAVE components (availability can be checked using d1_devicesupported, register access to unavailable hardware units is ignored).
