Texture registers

All texture registers are write only, reading will return undefined results.  Patterns and textures require a 1D or 2D index (UV Coordinate) at every pixel.  Specialized UV-Limiters are available to generate these.

Texture

D2_TEXORIGIN

Texture base address

Offset: 47

Address of the upper left corner texel.  Addresses inside the framebuffer can be used as valid texture origin as well.

D2_TEXPITCH

Texels per texture line

Offset: 45

Pitch is equal or bigger than texture width.

D2_TEXMASK

Texture size or texture address mask

Offset: 46

Depending on the clamping mode this register encodes the clamp limit or wrap mask. bits 31-11 V mask and bits 10-0 U mask

UV Limiters

D2_LUSTART

U Limiter start value

Offset: 36

The start value is a 16:16 fixedpoint number valid at the first pixel of the bounding box.

D2_LUXADD

U Limiter X-Axis increment

Offset: 37

The xadd value is the 16:16 fixedpoint difference between two samples with a distance of 1 pixel along the x axis.

D2_LUYADD

U Limiter Y-Axis increment

Offset: 38

The xadd value is the 16:16 fixedpoint difference between two samples with a distance of 1 pixel along the x axis.

D2_LVSTARTI

V Limiter start value integer part

Offset: 39

D2_LVSTARTF

V Limiter start value fractional part

Offset: 40

The start value is a 32:16 fixedpoint number valid at the first pixel of the bounding box.

D2_LVXADDI

V Limiter X-Axis increment integer part

Offset: 41

The xadd value is the 32:16 fixedpoint difference between two samples with a distance of 1 pixel along the x axis.

D2_LVYADDI

V Limiter Y-Axis increment integer part

Offset: 42

The xadd value is the 32:16 fixedpoint difference between two samples with a distance of 1 pixel along the x axis.

D2_LVYXADDF

V Limiter X and Y increment fractional parts

Offset: 43

Both fractional parts (for D2_LVXADDI and D2_LVYADDI) are stored in this register.  Y increment in the high word and X increment in low word.

CLUT

D2_TEXCLUT

Color Lookup Table for the indexed texture format 16x24bit

Offset: 54

Triggers a write into the CLUT if CLUT size is 16 x 24bit.

Bits[31..24]Index of the CLUT entry, that shall be written
Bits[23..0]Color Value RGB888

D2_TEXCLUT_ADDR

Color Lookup Table write address for the indexed texture format 256x32bit

Offset: 55

Start address for consecutive writes to D2_TEXCLUT_DATA if CLUT size is 256 x 32bit.

Bits[7..0]write address

D2_TEXCLUT_DATA

Color Lookup Table write data for the indexed texture format 256x32bit

Offset: 56

Writes one 32 bit color value into the CLUT if CLUT size is 256 x 32bit.  The write address is given by D2_TEXCLUT_ADDR and increments by 1 after each data write

Bits[31..0]Color Value ARGB8888 if CLUTFORMAT = argb8888
Bits[31..16], BITS[15..0]2x Color Value RGB565 if CLUTFORMAT = rgb565

see also D2_CONTROL2

D2_TEXCLUT_OFFSET

Offset to the texture index for the indexed texture formats i8, i4, i2 and i1

Offset: 57

The index offset is combined with the texture index by a bitwise OR operation.  For subbyte texture formats the CLUT can be divided into parts with different color sets.

Bits[7..0]index offset

Color keying

D2_COLKEY

Color key value

Offset: 58

The R, G and B components of the internal color representation of a texel is compared with the color key.  If the values are equal then A, R, G and B are set to 0 (transparent), else A is set to 1.

Color keying can be enabled by setting bit D2C_COLKEY_ENABLE in D2_CONTROL2

Bits[23..0]Color Key Value RGB888
